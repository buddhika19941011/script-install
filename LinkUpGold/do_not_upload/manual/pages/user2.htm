Registered users - Registration

The script which is used for users registration and also to modify users data is user.php. The signup form is displayed if you call the script without any variables, as user.php. It shows a form which is based on template user_join.html. Used variables:

#%in2%#

URL of your site

A field to enter Username. This part is based on template form_username.txt.

A field to enter Password. This part is based on template form_password.txt.

#%field_captcha_test%#

CAPTCHA image test. It can be visible/invisible, based on your configuration. This part is based on template form_captcha_test.txt. This is not visible if the form is used by a registered user.

The script automatically sends the new user an email with an ownerization code. The new user must click on the link which is included in this email to confirm that he/she is really the person who created the account. Unconfirmed users are deleted once a day.
The confirmation email is based on template user_confirm.txt. Please note that the first line of all email templates must begin with the word Subject: and below the line must be one blank line. This template contains this variable:

#%confirm_url%#

The confirmation URL.

When the new user clicks on the link in this email, he/she gets a page which is based on template user_confirmed.html. This page confirms that the account has been successfully confirmed and there is also a button to continue to users area. If you selected that you should receive an email when a new user joined, you now receive an email which is based on email template user_joined_edited_admin.txt.
