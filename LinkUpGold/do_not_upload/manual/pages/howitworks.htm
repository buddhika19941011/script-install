How it works

Most of the public pages is generated on-the-fly, the data is always fresh. Only your home page (index.php) and some parts of other pages are saved to static files, these files need to be created once a day. Click here to read a detailed info.

Styles

There are several styles available by default, each style has its own subdirectory in directory "styles". Each of these subdirectories contains a styles.css file where are defined all colors and fonts for that style. Each style also contains images, these are used for backgrounds of tables and pages. You can freely edit the styles.css files as well as images, you also can edit templates of each style or create your own style(s).

If you have a template and don't know how to use it in Link Up Gold, we can create a new style for you by using your own template.

Templates

All public pages are based on templates - these are normal HTML or text files editable in your favorite HTML or text editor. If possible, we recommend to use a plain text editor (Notepad, Crimson Editor, NoteTab, First Page 2000, HTML-Kit and so on) because WYSIWYG editors (like FrontPage, DreamWeaver etc.) often add unwanted tags or remove necessary items, this may cause an error. However if you use a WYSIWYG editor, make sure that your editor don't modify hidden fields of forms. Also note that the original templates are not complete HTML pages - the header and footer are included automatically by scripts (take a look below for more info) so if your editor adds header and footer tags (<html><header> .... </body></html>) to your templates, the finished pages will have these tags two times.

Most of the templates contain special tags which look like #%someword%#. Scripts replace these tags with some values (like Link Title, Description, numbers of clicks etc.). Templates can be found in directory styles/STYLE_NAME/templates. Styles which don't have their own templates or have only a few of them use "common" templates, these are stored in folder _common. In other words, when the script needs to create a page, it searches for the required template in the folder of the style which is just selected. If the required template is missing in this folder, it grabs it in folder _common.
Templates with html extension are always used to create complete HTML pages. Templates with txt extension are used for fragments of HTML pages (for example for a single link which should be repeated multiple times on a page).

HTML Templates

At the top of each public page there is automatically added content of the template _head1.txt. At the end of each public page is automatically added content of file _footer.txt. You can use these files to add some content to each public page. It may save you a lot of time when you will need to change something on all pages. By default, the _head1.txt is used to insert the first tags of each page.
In the "templates" directory is also one file which may be optionally inserted into each public page - _head2.txt. This file is by default used to show the header table and menu which is shared by all pages. It means that if you want to add your logo or something else to each page, this is the file to edit. Content of this file is not inserted automatically. It can be used in html templates. Use variable #%head%# to display this content.

There are also files whose content may be inserted into any public page to any place you want. Names of these files look like _inX.txt where X is a numeric value. You can change the amount of these files in configuration. The default value is 2. If you need more, create new files in the templates directory and enter the maximum allowed number of them in configuration. Since it may cause too high server load to use many of these files, we recommend to never use more than 10-15. If you want to insert a content of one of these files to any of your pages, write #%inX%# to any template HTML to the place where should be displayed this content. For example the variable #%in2%# shows the content of file _in2.txt. By default are these files used only for the tables on the left and right side of each page and for the search box at the top. In all these files _inX.txt may be used some variables which are automatically replaced with their values. These variables are:

#%site_url%#

URL of the directory

#%l_first_cats%#

List of links to all first level categories of links. Each line of this list is based on template _in_one_category.txt.

#%a_first_cats%#

List of links to all first level categories of articles. Each line of this list is based on template _in_one_category.txt.

#%v_first_cats%#

List of links to all first level categories of videos. Each line of this list is based on template _in_one_category.txt.

#%n_first_cats%#

List of links to all first level categories of news. Each line of this list is based on template _in_one_category.txt.

#%b_first_cats%#

List of links to all first level categories of blogs. Each line of this list is based on template _in_one_category.txt.

#%top_tags%#

The most popular keywords in links, articles, blogs, news and videos.

#%l_toprated%#

Top rated links - number of links in this list is editable in configuration. Each line of this list is based on template _in_one_item.txt.

#%l_popular%#

Most popular links (with the highest number of hits since the last monthly reset) - number of items in this list is editable in configuration. Each line of this list is based on template _in_one_item.txt.

#%a_toprated%#

Top rated articles - number of items in this list is editable in configuration. Each line of this list is based on template _in_one_item.txt.

#%a_popular%#

Popular articles - number of items in this list is editable in configuration. Each line of this list is based on template _in_one_item.txt.

#%v_popular%#

Most popular videos (with the highest number of reads since the last monthly reset) - number of items in this list is editable in configuration. Each line of this list is based on template _in_one_item.txt.

#%v_toprated%#

Top rated videos - number of items in this list is editable in configuration. Each line of this list is based on template _in_one_item.txt.

#%n_popular%#

Most popular news - number of items in this list is editable in configuration. Each line of this list is based on template _in_one_item.txt..

#%n_toprated%#

Top rated news - number of items in this list is editable in configuration. Each line of this list is based on template _in_one_item.txt.

#%b_popular%#

Most popular blogs - number of items in this list is editable in configuration. Each line of this list is based on template _in_one_item.txt.

#%b_toprated%#

Top rated blogs - number of items in this list is editable in configuration. Each line of this list is based on template _in_one_item.txt.

There are also some special variables which look like #_variable_#. Please don't edit these variables, these are usually used to show or hide a content of a page, depending on the current configuration and needs. These variables are needed for the system to work.

Email templates
All email templates may be found in styles/_common/email_templates directory. If you need different email templates for individual styles (for example to create a multi-language directory), copy the directory email_templates to directory of the style(s) where you want to use it. The system is the same as for templates for pages - the script searches for a template in directory of the selected style, if the required template does not exist, it grabs it from the _common directory. Note that different styles are not applicable to all email templates. The script can apply different styles to those email templates which are used to send emails from public pages (like link_added.txt, tell_friend.txt), it applies the currently selected style. Different styles are not supported for email templates sent by admin (for example link_approved.txt, newsletter.txt) because there is not a style selected in the time when the email is sent.
The first line of each email template must always begin with the word "Subject: " and below this line must be one blank line, then is a text of the email.
In configuration, there is the option to choose to send all emails in HTML or plain text format. You also can send only selected emails in HTML format. To do it, enter "HTML_EMAIL" (without quotes) at the end of subject line of the selected template.

Messages
Messages generated by scripts are used when the system needs to show some info to visitors. It may be for example error messages in the link submit form in the case that the entered data was rejected etc. The system is the same as for templates - you can have different messages for each style. By default, messages for style "_common" are used for all styles.

CSS Styles

Each of the color styles contains a styles.css file in its subfolder in styles directory. In this file are editable all colors, fonts and all other features of the color style. There also can be enabled/disabled the left and right columns. For example the left column is defined by using this code:

.left_column {
        display:block;
}

You can edit it like this to hide the left column:

.left_column {
        display:none;
}
