Registered users - Options in users area

The script which is used to show users area and also to modify users data is user.php. The login form is in template user_login_form.txt and is displayed at the top of all pages.

This script also offers a password reminder at user.php?action=user_remind. There is a form to enter email. This form sends an email with the lost password to this email address. The page is based on template user_remind.html, the email uses template user_user_remind.txt.

Once an user logged inside users area, it shows a page which is based on template user_home.html,. This page explains all the options available for registered users. There are available these variables:
 

#%in1%#
#%in2%# 

Content of files _in1.txt .... . This may be used to insert some information to several or all pages. In these files may be also used some variables. More info here.

#%head%#

Content of file _head2.txt. This may be used to insert some information to the begin of several or all pages. More info here.

#%site_url%#

URL of your site

All the options available for registered users are also in template _in1.txt, near top. These options are displayed for registered users only. These options are:

  • Edit your data
  • Create a new AdLink
  • Create a new link
  • Your links
  • Your orders
  • Purchase new funds
  • Create a new article
  • Your articles
  • Create a new blog
  • Your blogs 
  • Your bookmarks
  • Log off

Edit your data

An user edit form is available at user.php?action=user_edit. The page is based on template user_edit.html. This template contains these variables:

#%in1%#
#%in2%# 

Content of files _in1.txt .... . This may be used to insert some information to several or all pages. In these files may be also used some variables. More info here.

#%head%#

Content of file _head2.txt. This may be used to insert some information to the begin of several or all pages. More info here.

#%site_url%#

URL of your site

#%info%#

This is only used after editing the user info and it informs the user if the info has been save or if any error has occurred.

#%field_username%#

A field to modify Username. This part is based on template form_username.txt.

#%field_password%#

A field to modify Password. This part is based on template form_password.txt.

#%field_name%#

A field to modify Name. It can be visible/invisible, based on your configuration. This part is based on template form_field.txt.

#%field_email%#

A field to modify Email. It can be visible/invisible, based on your configuration. This part is based on template form_field.txt.

#%field_company%#

A field to modify Company. It can be visible/invisible, based on your configuration. This part is based on template form_field.txt.

#%field_address1%#

A field to modify Address line 1. It can be visible/invisible, based on your configuration. This part is based on template form_field.txt.

#%field_address2%#

A field to modify Address line 2. It can be visible/invisible, based on your configuration. This part is based on template form_field.txt.

#%field_address3%#

A field to modify Address line 3. It can be visible/invisible, based on your configuration. This part is based on template form_field.txt.

#%field_country%#

A field to modify Country. It can be visible/invisible, based on your configuration. This part is based on template form_field.txt.

#%field_phone1%#

A field to modify Phone number 1. It can be visible/invisible, based on your configuration. This part is based on template form_field.txt.

#%field_phone2%#

A field to modify Phone number 2. It can be visible/invisible, based on your configuration. This part is based on template form_field.txt.

#%field_styles%#

List of site styles to select from. This part is based on template form_no_field.txt.

#%field_pictures%#

Image upload fields. These images are then published on the user details page.

#%field_detail%#

A field to enter an article which is then visible on the public user page.

#%field_site_info%#

Fields to modify site URL and Site title. It can be visible/invisible, based on your configuration. This part is based on template form_field.txt.

#%field_newsletters%#

List of available newsletters. You can configure newsletters in Administration => Main configuration. This part is based on template form_field.txt.

Create a new link

This option is described here.

Your links

This link shows a page which uses template links_for_owner_list.html. Each line in the list of links uses the template links_for_owner_list.txt. It offers a link to each link details page, a link to link edit page, a link to a page with detailed statistic and if you allow users to delete their links, it also contains a "delete link" link.

Your AdLinks

This link shows a page which uses template user_adlinks.html. Each line in the list of AdLinks uses the template user_adlink.txt. It offers the option to view and edit the AdLinks and also the option to order new clicks. More info about AdLinks can be found here.

Your orders

This link shows a page which uses template user_orders.html. Each line in the list of orders uses the template user_order.txt.

Purchase new funds

This link lets users to add new funds to their account. These funds can be then used to add clicks, impressions, days to advertising links and to add clicks to an AdLink. More info about the payment system is here.

Create a new article

This option is described here.

Your articles

This link shows a list of articles added by the user. The list is based on template article_edit_list.html. This list contains links to article details page, link to edit form and if you allow users to delete their articles, it also shows a "delete article" link. Each of the lines is based on template article_edit_list.txt.

Create a new blog

This option is described here.

Your blogs

This link shows a list of articles added by the user. The list is based on template blog_edit_list.html. This list contains links to blog details page, link to edit form and if you allow users to delete their blogs, it also shows a "delete blog" link. Each of the lines is based on template blog_edit_list.txt.

Your bookmarks

This link shows links, articles and articles added that have been added to the user favorites list. The option to add items to this list is in categories, search results and link/article detail pages. The list of favorites inside users area contains title of each of these items linked to the item details page and the option to remove it from this list. The list uses template user_favorites.html.

Log off

This option is available at user.php?action=user_log_off. An user who clicked to this link is logged off and a page which is based on template user_logoff.html is displayed.
