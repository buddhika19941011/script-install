Article submission - article_create.php

This script is used to submit a new article. his script is fully configurable, all the options are available in Administration => Public form => Articles. You can choose visible and required fields, you can set how many images can be uploaded, you also can set size of images and thumbnails. There are many other options, incl. the option to choose that only registered users can add articles.

The submit form can be reached at article_create.php, in this case there are available all categories to choose from. It also can be reached at article_create.php?c=CATEGORY_NUMBER. If you use this URL, there is not available the list of categories, the form can be used only to create an article in a selected category. The links in this shape are available by default in the template category.html.

The submit form uses template article_submit.html, these variables are available:

#%in2%#

#%head%#

Content of file _head2.txt. This may be used to insert some information to the begin of several or all pages. More info here.

#%site_url%#

URL of your site.

#%info%#

If someone submitted the form but the system rejected it for some reason, it displays the error which occurred.

#%field_title%#

A field to enter Title. It can be visible/invisible, based on your configuration. This part is based on template form_field.txt.

#%field_description%#

A field to enter Subtitle. It can be visible/invisible, based on your configuration. This part is based on template form_field.txt.

#%field_text%#

A field to enter Text of the article. It can be visible/invisible, based on your configuration. This part is based on template form_detail_textarea.txt or form_detail_html.txt, depending on your configuration.

#%field_categories%#

A field to select Categories . This part is based on template form_no_field.txt.

#%field_pictures%#

A field to upload Pictures. The number of these fields is editable by admin. Each of them is based on template form_upload.txt.

#%field_user_defined%#

Fields to enter User Defined Items (you can define them in Configuration). This part is based on templates form_user_item_text.txt, form_user_item_textarea.txt, form_user_item_htmlarea.txt, form_user_item_checkbox.txt, form_user_item_radio.txt or form_user_item_select.txt, depending on kind of each of these items.

#%user_item_28%#

If you, for some reasons, don't want to display all available User Defined Items, use this variable to display each individual item. This variable is based on templates form_user_item_text.txt, form_user_item_htmlarea.txt, form_user_item_textarea.txt, form_user_item_checkbox.txt, form_user_item_radio.txt or form_user_item_select.txt, depending on the type of the selected field.
This is only an example, replace the number 28 by real number of the item you want to display. These numbers can be found in Configuration => User Defined Items.

#%user_item_name_28%#

You can use this variable to display label of an individual user item.

#%user_item_value_28%#

You can use this variable to display edit field for an individual user item.

#%field_keywords%#

A field to enter keywords. This part is based on template form_keywords.txt.

#%field_dates%#

A field to enter start and end date. It can be visible/invisible, based on your configuration. This part is based on template form_dates.txt.

#%field_name%#

A field to enter Name. It can be visible/invisible, based on your configuration. This part is based on template form_field.txt. This is not visible if the form is used by a registered user.

#%field_email%#

A field to enter Email. It can be visible/invisible, based on your configuration. This part is based on template form_field.txt. This is not visible if the form is used by a registered user.

#%field_password%#

A field to enter Password. It can be visible/invisible, based on your configuration. This part is based on template form_password.txt. This is not visible if the form is used by a registered user.

#%field_captcha_test%#

CAPTCHA image test . It can be visible/invisible, based on your configuration. This part is based on template form_captcha_test.txt.

Once an article has been submitted, multiple states can occur, depending on your configuration. These are:

  • You selected that it has to inform admin when a new article has been submitted. In this case you receive an email by using template article_admin.txt.
  • You selected that it has to send an email to article owner immediately when a new article has been submitted. In this case article owner receives an email by using template article_added.txt.
  • You selected that new articles must be reviewed by an admin. In this case the article is waiting in Administration => Queue. Once you approved it, the article owner receives an email based on template article_approved.txt.
  • You selected that new articles are immediately online. In this case is the article instantly visible on public pages.
