HTML Plugin - static HTML pages

If this option is used, the plugin saves most public pages to static HTML files. It can be useful if you have a high traffic site - PHP scripts need more server resources than static HTML files. By using this option, your pages are displayed faster.

The following pages become static:

  • Your home page.
  • All categories. Names of the files are by default created from names of categories. You also can set names of files of a selected category manually in the category edit form - there is the field "Name of static html page".
  • It can optionally create a page for each individual link and article. The names of files are automatically created from the titles of the items. You also can manually enter file names in the link and article edit forms - there is the field "Rewrite URL" for it.

In Configuration there are a few fields that need to be filled in if you want to use this option. The required fields are these:

  • Extension of created static pages.
  • Name of your home page.
  • Folders to store pages (each of these fields should have an unique value).

If you use this option, there are some limitations because your pages are static and can't respond immediately to all actions of your visitors. Below is a list of these limits:

  • If someone clicks on a link, article, blog, video or news item the number of clicks/reads will not increase immediately but after the next daily rebuild.
  • If PHP on your server is running in a "safe mode", you can't use this feature because the script needs some time (several minutes) when rebuilding pages and in "safe mode" the server doesn't allow it.
  • You can't use the automatic rebuild function. Set a Crontab or Task Manager job to rebuild your pages or do it manually in the Administration.
  • You can't use the "Select style" feature which allows users to select the style they like, only one style can be used.
  • CSS styles are taken from the style you selected in Configuration but all templates and messages are always taken from directory "_common".
  • The field "Use Ajax to load individual pages in categories" must be checked.
  • You can't sell impressions for advertising links as the pages are static so the script can't detect each impression. For this reason you can't sell them and also users can't see them in statistics. However you can still sell clicks or days.
  • New RSS content can be available after the daily rebuild only.
  • The front page with a header and search form only is not available.
     
Follow these steps to enable the option to create static pages:
  • Check field "Build static html files" in Configuration.
  • Enter values to all the required fields (see above).
  • Set permission of your home directory to 777
  • If you set to create category pages and detail pages in another directories, create these directories and chmod them to 777.
  • If you want to use Crontab or Task Manager to rebuild your pages, set the Daily job as described here. Note that you can't use the automatic rebuild feature but you still can rebuild your pages manually in any time in the Administration.
  • If you set to create a static page for each individual article, enter the following HTML code to template article_details.html. It lets you track number of reads, even when this page is static:
    <img border="0" width="1" height="1" src="#%site_url%#/article.php?n=#%n%#&track_hit=1">
  • If you set to create a static page for each individual video, enter the following HTML code to template video_details.html. It lets you track number of views, even when this page is static:
    <img border="0" width="1" height="1" src="#%site_url%#/video.php?n=#%n%#&track_hit=1">
  • If you set to create a static page for each individual news item, enter the following HTML code to template new_details.html. It lets you track number of reads, even when this page is static:
    <img border="0" width="1" height="1" src="#%site_url%#/new.php?n=#%n%#&track_hit=1"> 
  • If you set to create a static page for each individual blog, enter the following HTML code to template blog_details.html. It lets you track number of reads, even when this page is static:
    <img border="0" width="1" height="1" src="#%site_url%#/blog.php?n=#%n%#&track_hit=1">      
  • Use function Reset/rebuild => Daily job in the Administration to rebuild pages, then check if all works well.
